# Conventions

This directory contains per-language conventions and documentation standards.

## Contents

- [documentation.md](./documentation.md) - How to write documentation for this project

## Purpose

Convention files define the standards that all contributors should follow:
- Code style and formatting
- Documentation structure and style
- Language-specific best practices

## What Goes Here

- Language-specific coding conventions (e.g., `rust.md`, `python.md`)
- Project-wide standards (e.g., `general.md`)
- Documentation guidelines (e.g., `documentation.md`)

## What Doesn't Go Here

- Design rationale (belongs in [design/](../../design/))
- User guides (belongs in [guides/](../../guides/))
- Task definitions (belongs in [tasks/](../../tasks/))
