# Agent Design

This directory contains design decisions and rationale for the agent component.

## Contents

- **[Architecture](architecture.md)** - High-level architecture and component responsibilities

## Purpose

Agent design documentation explains **WHY** decisions were made about:
- Agent architecture and component structure
- Context management strategies
- Cost control mechanisms
- Sandboxing approach

## What Goes Here

- Architecture decisions for the agent
- Design rationale for agent behavior
- Trade-offs in agent implementation
- Cross-cutting concerns affecting agent design

## What Doesn't Go Here

- Implementation details (belongs in source code docstrings)
- Task definitions (belongs in [tasks/](../../tasks/))
- User guides (belongs in [guides/](../../guides/))
