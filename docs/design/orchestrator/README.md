# Orchestrator Design

This directory contains design decisions and rationale for the orchestrator component.

## Contents

- [architecture.md](./architecture.md) - High-level architecture and design principles
- [protocol.md](./protocol.md) - Agent-orchestrator communication protocol
- [environments/](./environments/) - Individual environment design decisions

## Purpose

Orchestrator design documentation explains **WHY** decisions were made about:
- Orchestrator architecture and component structure
- Environment protocols and interfaces
- Command processing pipeline
- State management

## What Doesn't Go Here

- Implementation details (belongs in source code docstrings)
- Task definitions (belongs in [tasks/](../../tasks/))
- User guides (belongs in [guides/](../../guides/))
